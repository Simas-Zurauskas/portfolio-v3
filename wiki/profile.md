## Pitch Assessment Guide

When evaluating whether a pitch/opportunity is a good fit, check against these criteria:

### Strong fit signals

- Full-stack web (React/Next.js + Node.js) — this is the core
- React Native mobile apps — extensive production experience
- AI integration (LLM chatbots, RAG, multi-agent systems) — actively building in this space
- SaaS products — has built multiple from scratch
- Real-time features (WebSockets, live tracking, multiplayer) — proven at scale
- Startup/founder context — comfortable with ambiguity, used to wearing multiple hats
- Projects needing end-to-end ownership — this is the default working mode
- Industries already covered (InsurTech, Logistics, FinTech, HealthTech, EdTech) — domain knowledge exists

### Moderate fit — can deliver but not the deepest experience

- Pure backend / infrastructure-heavy projects (no frontend) — can do it, but frontend is stronger
- Design-only work (Figma/UI design without development) — does it as part of full-stack, not standalone
- WordPress / PHP development — not in the current stack
- DevOps-heavy (Kubernetes, Terraform, complex cloud infra) — Docker intermediate, not a cloud architect
- Data engineering / ML model training — integrates AI, doesn't build models from scratch

### Weak fit signals

- Java, C#, Python-heavy backends — JavaScript/TypeScript ecosystem only
- iOS/Android native (Swift/Kotlin) as primary stack — React Native, not native
- Low-code / no-code platforms — not the working style
- Hardware / embedded systems — no experience
- Game development (Unity, Unreal) — not applicable
